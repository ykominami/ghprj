# サブコマンド search
## 動作
引数で指定された条件に合致する、すべてのリポジトリについての情報を配列で返す。
ただし、デフォルトでは、リポジトリ名の配列を返す。
オプション引数 `---all` が指定された場合は、リポジトリ情報のすべての項目を含む配列を返す。

## 必須第１引数 <search_name>
### 値
- `public` visibility が public のリポジトリのみを検索対象にする
- `private` visibility が private のリポジトリのみを検索対象にする
- `both` visibility が public と private のリポジトリのみを検索対象にする
- `internal` visibility が internal のリポジトリのみを検索対象にする
- `latest10` visibilityに関係なく、存在するリポジトリのうち、直近に登録された最大10個までを検索対象にする（10個未満しか存在しなければ、すべてを検索対象にする）

## オプション引数
### `--name`
- 型: str
- デフォルト値: None
- 説明: リポジトリ名の部分文字列パターン
- 動作: リポジトリ名が指定された部分文字列パターンに合致する、リポジトリを取得する
### `--user`
- 型: `str`
- デフォルト値: `None`
- 説明: GitHub ユーザー名
### `--verbose`
- 型: フラグ
- デフォルト値: `False`
- 説明: 詳細出力
### `--all`
- 型: フラグ
- デフォルト値: `False`
- 説明: リポジトリ情報のすべてを返す
