# サブコマンド search
## 動作
必須第１引数 search_name の値が `latest10` の場合を除き、必須第１引数の値と、指定されたすべての検索条件オプション引数に対応する条件を満たす、すべてのリポジトリに対して、リポジトリ情報のうちの、リポジトリ名のみを取得して、JSON形式の配列で返す。
ただし、出力情報オプション引数 `--all` が指定された場合は、リポジトリ情報のすべての項目を含むJSON形式の配列を返す。

必須第１引数 search_name の値が `latest10` の場合は、検索条件オプション引数(`--name` と `--user`)の指定は無視する。

いかなる引数の指定方法であっても、同一の必須引数、またはオプション引数が複数個指定された場合は、エラーとする。
エラーの判定は、引数の規定の中では、最も優先度が高い。

正常系のオプション引数同士には、順序による意味の違いはない。

## 必須第１引数 search_name
### 値
- `public` visibility が public のリポジトリのみを検索対象にする
- `private` visibility が private のリポジトリのみを検索対象にする
- `both` visibility が public と private のリポジトリのみを検索対象にする
- `internal` visibility が internal のリポジトリのみを検索対象にする
- `latest10` visibilityに関係なく、存在するリポジトリのうち、直近に登録された最大10個までを検索対象にする（10個未満しか存在しなければ、すべてを検索対象にする）

## 検索条件オプション引数
### `--name`
- 型: str
- デフォルト値: None
- 説明: リポジトリ名の部分文字列パターン
- 動作: リポジトリ名が指定された部分文字列パターンに合致するリポジトリを取得する。
### `--user`
- 型: `str`
- デフォルト値: `None`
- 説明: GitHub ユーザー名。
- 動作: 所有者が合致するリポジトリを取得する。
## 詳細出力オプション引数
### `--verbose`
- 型: フラグ
- デフォルト値: `False`
- 説明: 詳細な内容を出力する。デバッグ目的に使うことを想定している。
## 出力情報オプション引数
### `--all`
- 型: フラグ
- デフォルト値: `False`
- 説明: リポジトリ情報のすべてを返す
